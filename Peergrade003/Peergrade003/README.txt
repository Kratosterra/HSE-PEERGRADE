Добро пожаловать в исходный код “Файлового менеджера”

 ----------------------------------------------------------------------------------------------

Кода разделен на логические модули - файлы:
 1)Program.cs - содержит скелет программы (основной файл). 
 2)Environment.cs - содержит элементы интерфейса программы.
 3)Actions.cs - содержит методы для выполнения каждого действия, запрошенного пользователем. 
 4)Input.cs - содержит класс с методами для проверки корректности ввода пользователя.
 5)ManageTools.cs - содержит методы, которые выполняют вспомогательную роль в работе программы .

 ----------------------------------------------------------------------------------------------

~Полезные функции~

1) Дополнение пути по нажатию TAB.
2) Отмена любого действия при вводе ?END? при вводе любого пути.

~Какие ограничения имеет программа~

1) Ограничение на возможность названия файлов пустым названием.
2) Поддерживается создание и чтение файлов только в формате .txt.
3) Размер файла для чтения, конкатенации или нахождения различий не должен превышать 256 мб.
4) Конкатенация поддерживает от 2 до 10 файлов.

~Скорость работы~

Скорость работы программы во время поиска файлов сильно зависит от особенностей системы, 
в некоторых сценариях расширенный поиск по маске может происходить около минуты
при применении общей маски на основной диск системы.

~Какие кодировки поддерживаются?~
(проверить кодировки можно в notepad++)

1)UTF-8
2)UTF-16
3)ASCII
4)UTF-32 (не поддерживается notepad++)

~Визуальные проблемы с TAB на некоторых версиях Rider.~

В некоторых версиях Rider при использовании клавиши TAB поле ввода переносится на следующую строку,
это не влияет на работу программы, но создаёт неприятное впечатление,
поэтому рекомендуется запускать программу в VisualStudio или из скомпилированного .exe. 
Также стоит предупредить пользователей MAC, что у вас могут возникнуть такие же визуальные баги,которых нет на Windows.

Нажатием на Enter вы всё еще можете принять текущие введенные данные и свободно продолжать работу с программой.

~Особенности записи в файлы~

Запись текста в файлы осуществляется одной строкой, для задания требуемой кодировки.

~Особенности копирования и перемещения~

При копировании файла в ту же директорию где он сейчас, будет создаваться его копия с другим названием,
например для *.txt создаётся копия (номер) *.txt. 
При перемещении файла в ту же директорию где он сейчас - ничего не происходит.

~Копирование файлов по маске~

При вводе пути до несуществющией  папки создаётся не только конечная несуществующая папка но и все папки,
указанные в пути, которые не существуют.  

~Перехват потока~

При открытии файла в приложениях, захватывающих файл в поток чтения/записи,
программа не сможет считать данные из файла и вернет ошибку.
